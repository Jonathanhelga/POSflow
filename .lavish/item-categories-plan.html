Item Categories

Let users tag inventory items with a category, edit it later, and group the POS grid by category so items are faster to find. Everything below was verified against your current code.

Goal & current state

Goal: Add a category to each item, editable after creation, and add a "Category" grouping to the existing Sort bar.

✓ verified in code

  • Items are created in the Add-Item modal; add_item_ui.js builds formData and submitItemData() spreads it straight into /inventory. No category field exists today.

Decision 1 — How are categories defined?

What a "category" actually is: just a label that groups similar items, like the aisles in a shop — "Drinks", "Snacks", "Stationery". Your question is really: who decides that list of labels?

The unlock — it isn't either/or. The recommended option is a hybrid: you type your own category (so you never have to plan them in advance), and the app remembers each one and suggests it next time. So it's as flexible as free text, but stays as tidy as a fixed list — you won't end up with "Drinks", "drink", and "DRINKS" as three separate things.

A · You type them, app remembers recommended

A text box that autocompletes from categories you've already used (via <datalist>). Type a brand-new one anytime; reuse old ones with one tap.

Cost: tiny. No new collection, no setup screen. Flexible and stays consistent.

B · Managed fixed list

A separate place to define the allowed categories, then pick from a dropdown only.

Cost: higher. Needs a categories collection + a management UI. Cleaner data, less flexible for a fast-moving bazaar.

Decision 2 — Where does "Sort by Category" go?

You pictured this inside Manage Item, but in the real code the Sort bar is on the main POS screen. Two ways to honour your intent:

A · Add to the main Sort bar recommended

Adds "Category" next to Color / Name / Price / Stock, grouping the POS grid you actually sell from.

Cost: tiny. Matches how sorting already works.

B · Also add a filter inside Manage Item

A category filter in the Manage Item modal too, for back-office editing.

Cost: a bit more. Can be a fast follow-up after A.
